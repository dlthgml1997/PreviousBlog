  • 자식 클래스 = 파생 클래스 = 확장된 클래스 = 하위 클래스

    상속을 통해 소스코드의 재사용성이 보장되며 가독성이 높아진다.

  • 공통된 기능을 갖는다고 모두 상속할 수 있는 것은 아니다. 부모 클래스와 자식 클래스의 관계가 일반화와 특별화의 관계, ‘is a~’관계 에 있어야 한다.

    • 문법적으로 강제되는 것이 아닌 논리적으로 ‘is a~’관계가 성립되어야 한다.

      예) 프린터가 부모 클래스일 때 도트 프린터, 잉크젯 프린터, 레이저 프린터는 프린터라는 공통적인 특징을 가지기 때문에 상속 관계가 성립될 수 있지만 N사 카메라와는 기능이 같더라도 상속 관계가 성립될 수 없다.

    • 부적절한 상속은 소스코드에 대한 분석과 개발을 어렵게 만든다.

  • 상속은 부모로부터 물려받은 특징에 자식 클래스에서만 가지는 추가적인 특징을 더해서 만들어진 것이다.

  • 자바는 개발의 편의성과 가독성을 위해 문법적으로 단일 상속만을 허용한다.

    • 단일 상속이란, 하나의 클래스는 오직 하나의 부모 클래스만 상속할 수 있다는 것을 의미한다.
    • 다중 상속은 동일한 변수와 메서드를 상속 되는 문제가 발생한다.

2. 메서드 Overloading과 Overriding

메서드 오버라이딩 (메서드 재정의)

상속 관계에 있는 두 개의 클래스에서 부모 클래스의 메서드를 재사용하지 않고 새롭게 정의하여 사용하는 것이다.

  • 자식 클래스에서 재정의된 메서드는 부모 클래스의 메서드와 메서드 이름, 매개변수의 유형과 개수가 동일해야한다. (동일한 시그니처를 가져야 한다.)

  • 예제

    • super 예약어를 사용해 메서드를 overriding 하면서 부모 클래스의 메서드를 사용할 수 있다.

  • final 예약어와 Overriding

    • final + 변수 => 상수를 의미한다.

    • final + 클래스 => 자식 클래스를 가질 수 없는 클래스임을 의미한다.

    • final + 메서드 => 오버라이딩을 금지하는 것을 의미한다.

      • 자식 메서드가 부모 메서드를 오버라이딩할 때, 부모 메서드에 존재하는 메서드를 호출하지 않으면 문제가 발생한다.

      • 이를 방지하는 방법이 final 예약어를 사용하는 것이다.

메서드 오버로딩

하나의 클래스에 동일한 이름의 메서드가 여러 개 중복 정의 되어 있는 것이다.

  • 부모 클래스와 메서드 이름이 동일해야 하며, 매개변수의 개수나 타입 중 최소 하나를 다르게 하여 재정의하는 것이다.

오버라이딩은 메서드를 새롭게 정의하는 것 ! 오버로딩은 동일한 이름의 메서드를 여러 개로 정의하는 것!

[백준알고리즘] 1065: 한수 (파이썬)

2021-01-11
  • 문제링크: https://www.acmicpc.net/problem/1065

풀이

  • 한수란?
    각 자리 수가 등차 수열을 이루는 자연수 하나를 말한다.
    여기서 등차 수열은 문제에도 나와있다 싶이 연속된 두 개의 수의 차이가 일정한 수열이다.
    일단 1부터 9는 모두 한수이다. 11은 각 자리의 차가 0으로 동일하기 때문에 동차가 0인 한수이다. 111도 마찬가지이다. 동차가 1인 한수로는 123, 345 등이 있고, 동차가 2인 한수로는 135, 468 등이 있다.

첫번째 풀이 (N이 1000 이하인 조건을 만족하는 풀이)

  • 99까지는 모두 한수이기 때문에 n이 99 이하이면 무조건 n을 출력하고 실행을 종료하도록 구현했다.
  • n 이 123이면 nums 는 [1,2,3] 이 된다.
  • 세자리 부터 999까지는 두번째자리수와 첫번째 자리수의 차이값이 세번째 자리수와 두번째 자리수의 차이값 과 같다면 answer에 1을 더하는 방식으로 구현했다.

두번째 풀이 (N이 1001 이상이어도 구할 수 있는 코드)

  • check 함수는 각자리수를 담은 nums 배열을 받는다.
  • check 함수는 n이 100이상일때만 실행되기 때문에 반드시 nums[1]과 nums[0]이 존재한다. 이를 이용해 동차를 구했다.
  • nums 배열의 마지막 자리까지 차이를 구하며 동차와 다른 즉시 0을 반환하고 이를 answer에 더한다.
  • 만약 모든 자리수의 차가 동차와 같다면 1을 반환해 이를 answer에 더한다.

[백준알고리즘] 7576: 토마토 (파이썬)

2021-01-10
  • 문제 링크: https://www.acmicpc.net/problem/7576

풀이

시간 초과의 늪에 빠졌던 문제다. 한 박스에 익은 토마토가 2개 이상일 수 있는 점을 유의해서 bfs를 구현하는 점이 어려웠다.

  • 먼저 첫 날, 익은 토마토(1)의 좌표를 모두 apples에 담는다.

  • bfs를 실행한다.

    • day 를 -1 로 초기화한 이유

      1. 먼저 예외 상황을 생각해보자. 만약 모든 박스가 1과-1로 채워져 있는 경우에는 주변을 익게 하지 않고도 모든 사과가 익어있는 상황이기 때문에 0이 프린트 되어야한다. 그러나 apples에는 1의 좌표가 모두 담겨있기 때문에 bfs 함수의 while문이 한번은 실행될 것이다. 그렇기 때문에 day는 -1로 초기화 해준다.

      2. 모든 사과가 익은 날에 마지막 익은 사과는 자신 주변에 안익은 사과가 없어도 apples에 좌표가 담긴다. 그렇기 때문에 마지막 while문이 실행되고 이때 day 가 + 1 되기 때문에 처음에day를 -1로 초기화해야 정상적인 day 가 출력될 수 있다.

    • 하루마다 전날 익은 사과는 apples에서 비우고 새롭게 익은 사과들을 apples에 담기도록 구현하기 위해 apples의 길이만큼 for문을 실행한다.

    • 모든 1주변 사과를 익게 만든 후, 안익은 사과가 있는지 확인하고 안익은 사과가 있다면 -1을 출력한다.

중요 포인트 !

  • 기존에 사용해왔던 queue = [] , queue.pop(0) 방식으로 하면 O(n)으로 느려지기 때문에 collections의 deque로 구현해 O(1) 이 되도록 해야한다.

    • 하루마다 while 문이 실행되도록 해서 이중 포문을 실행하는 거부터 문제다 !
      • 하루마다 익은 토마토 주변 토마토를 익게하고 visited에 몇일 지났는지를 표시한다.
      • 그 다음 날, while 문이 실행될 때 visited가 전날인 토마토를 찾아 다시 주변 토마토를 익게하도록 했다.
    • while 문 종료 조건은 이중 포문을 다 돌았을 때, 한번도 주변 토마토를 익히지 않았을 때이다. (익힐 수 있는 토마토는 다 익힌 상태)
    • 첫날에 안익은 토마토 주변이 다 비어있으면 -1을 print하고 실행을 종료하는 check 함수를 구현했다.

Java Programming - 객체지향 핵심 원리 (자바 Modifier와 생성자)

학습목표

  2. 생성자의 기능 이해
  3. this 예약어의 의미와 활용 방법 학습

1. 자바의 Modifier

static

멤버 변수와 메서드 앞에 붙일 수 있는 modifier로서, 활용 방법을 제어함.

  • static 예약어가 붙지 않는 인스턴스 변수

    • 생성된 인스턴스마다 그 안에 클래스의 인스턴스 변수들이 포함된다.
    • 일반적인 멤버 변수를 인스턴스 변수라고 부른다.

  • static 예약어가 붙는 클래스 변수

    • 클래스로부터 생성된 인스턴스에 포함되지 않는 변수

    • 많은 인스턴스를 생성하더라도 메모리에 하나의 변수만 존재한다.

    • 객체를 생성하지 않고도 접근할 수 있는 변수

    • 가장 대표적인 클래스 변수로는 System 클래스의 out 변수가 있다.

  • 클래스 변수와 인스턴스 변수의 차이

    • 인스턴스 변수는 클래스로부터 객체가 생성될 때마다 객체의 변수들이 생성되지만 클래스 변수는 클래스로 부터 생성된 모든 객체들이 하나의 클래스 변수를 공유한다. 그렇기 때문에 클래스 변수는 객체가 생성될 때 메모리 영역을 할당하지 않고, 클래스가 로딩되는 과정에서 메모리에 한 번만 할당되는 멤버이다.
    • 각 객체가 동일한 값을 가지는 인스턴스 변수를 포함하면 메모리 관리에 악영향을 끼친다.
    • 클래스 변수는 값이 변경되는 경우, 클래스로부터 생성된 모든 객체에서 변경된 값을 사용할 수 있다.

  • 예시

    • 세 직원의 이름과 나이는 다르지만 회사이름은 S전자로 동일한 것을 볼 수 있다. 이때, 생성된 객체가 많을 수록 메모리를 불필요하게 사용하고 있다.

      이 문제를 클래스 변수로 해결할 수 있다.

    클래스 변수를 선언한 경우를 그림으로 표현하면 아래와 같다.

  • 클래스 메서드

    • 인스턴스를 생성하지 않고도 사용이 가능한 메서드

    • 대표적인 클래스 메서드로는 Integer클래스의 parseInt() 가 있다.

    • 클래스 메서드는 인스턴스 변수를 참조할 수 없다. (클래스 메서드는 인스턴스를 생성하지 않고 호출이 가능하지만, 인스턴스 변수의 경우 인스턴스를 생성해야만 메모리가 잡히기 때문이다.)

      즉, 클래스 메서드에서는 클래스 메서드 호출 할 수 있다.


final

  1. 변수

    final이 붙은 변수는 단 한 번 초기화가 가능하지만, 초기화 이후에는 값을 변경할 수 없는 상수이다.

    • 공통된 값을 가지는 변수를 클래스 변수로 선언하여 메모리를 효율적으로 사용할 때, 클래스 변수의 값을 변경할 수 있도록 허용하는 경우엔 static 예약어의 의미가 상실되므로, final 예약어를 결합하여 사용함으로써 초기화 된 값을 변경할 수 없게하는 것이 좋다.

    • final과 static 을 결합해 멤버 변수를 선언한 경우 이 상수의 이름은 대문자로 선언하고 여러 단어가 결합된 경우라면 단어와 단어 사이 ‘_’를 이용한다.

      예: COMPANY_NAME

  2. 메서드

    • final 예약어는 상속 구조에서 Overriding(메서드 재정의)를 금지하기 위해 사용될 수 있다.
      • 오버라이딩 : 상속 구조에서 부모 클래스의 메서드와 동일한 모양의 메서드를 자식 클래스에서 상속 받지 않고 새롭게 정의하는 것을 말한다.

  3. 클래스

    • 클래스 선언 시, 클래스 앞에 final 예약어를 추가하면 상속을 금지하는 의미이다.


abstract

클래스 선언부에서 abstract 사용 시 -> 추상 클래스

메서드를 선언할 때 abstract 사용 시 -> 추상 메서드

  • 추상 메서드

    메서드의 시그니처(리턴 타입, 메서드명, 매개변수)만 정의되고 구체적인 행위, 즉 블록({})부분은 정의되지 않는 메서드를 의미한다.

    • 메서드의 시그니처가 선언되어 있기 때문에 메서드의 기본적인 기능은 유추가 가능하다.

  • 추상 클래스

    추상 메서드를 포함하고 있다면 반드시 추상 클래스로 선언되어야 한다. 그러나 추상 클래스가 꼭 추상 메서드를 포함하고 있지 않을 수도 있다.

    • 추상 클래스와 상속의 결합

      • 상속 관계에서 부모 클래스가 추상 클래스라면 자식 클래스는 부모 클래스의 추상 메서드를 재정의(Overriding) 해야한다.


2. 생성자(Constructor)

생성자는 클래스로부터 객체를 생성할 때 호출되며, 객체의 멤버 변수를 초기화하는데 사용되는 메서드이다.

  • 생성자를 이용하여 생성된 객체의 멤버 변수에 적절한 값들을 설정할 수 있다. (코드 단순화)

  • 생성자의 특징

    • 반환형이 없다(void도 허용되지 않는다).

    • 생성자는 클래스와 같은 이름을 가지는 특별한 메서드이다.

    • 생성자는 이름은 같지만 매개변수를 다르게하여 여러 개를 중복 정의(Overloading)할 수 있다.

    • 생성자는 키워드 ‘new’와 항상 같이 사용된다.

      new 생성자()

    • 명시적으로 작성하지 않을 경우, 기본 생성자가 제공된다.

      • 기본 생성자: 매개변수가 없는 생성자로 일정한 규칙에 의해 자동 초기화된다.


3. this

생성자나 메서드의 매개변수 이름이 객체 변수의 이름과 같을 때 this 예약어를 사용해 이를 구별한다.

  • this 예약어 사용으로 멤버 변수와 매개변수 이름을 구분할 수 있고, 가독성을 높인다.

    • public : 모든 클래스에서 접근이 가능하다.

    • syncronyzed: Thread의 동기화를 위한 메서드이다.

    • 서로 다른 컴퓨터 상의 객체들도 원격으로 호출하여 실행할 수 있는 원격 메서드 호출과 관련된 RMI(Remote Method Invocation)기능의 라이브러리 제공

  4. 인터프리터에 의해 실행됨

    .java 형태의 텍스트 파일 -> 컴파일러에 의해 자바가상머신에서 수행될 수 있는 중간코드로 컴파일된다. -> .class(클래스 파일 == 바이트 코드) -> 실제 실행을 위해 인터프리터에 의해 기계어로 변환 -> 컴퓨터에서 실행

    자바는 컴파일 언어임과 동시에 인터프리터 언어이다.

  5. 견고

    에러의 원인이 되는 요소들을 제거

    • 포인터를 사용하지 않음
    • 가비지 컬렉션
    • 데이터 타입 검사를 통해 에러 초기 검사
    • 실행시간에 발생하는 에러 초기

  6. 안전

    • 컴파일 시에 엄격하게 데이터 타입을 검사함으로써 프로그램 실행 시 발생할 수 있는 비정상적인 상황을 미리 감지

  7. 플랫폼 독립(Architecture Neutral)

    사용하는 운영체제나 CPU 등의 하드웨어 사양에 관계없이 실행될 수 있음.

    이식성이 높다.

  8. 높은 성능

    가비지 컬렉션 : 메모리가 할당된 객체의 상태를 추적 관리, 더이상 참조 되지 않으면 다종으로 할당된 메모리 해제 (불필요한 메모리 낭비 방지)

  9. 멀티스레드

    스레드 : 독자적으로 수행하는 하나의 작은 프로그램

    멀티스레드: 한번의 여러개의 스레드가 동시에 수행되는 과정 / 하나의 CPU가 여러 개의 프로그램을 동시에 수행하도록 함 (수행 속도를 빠르게 함)

  10. 동적

    변화하는 환경에 잘 적응되도록 설계됨

    기존의 프로그램에 영향을 주지 않고, 라이브러리에 새로운 메서드나 속성들을 추가할 수 있음

    프로그램과 프로그램에서 사용되는 라이브러리와의 연결을 실행시간에 수행함

    -> 라이브러리의 변화를 곧바로 적용할 수 있음

[SW Expert Academy] 2043. 서랍의 비밀번호 (파이썬)

2020-12-30

문제

서랍의 비밀번호가 생각이 나지 않는다.

비밀번호 P는 000부터 999까지 번호 중의 하나이다.

주어지는 번호 K부터 1씩 증가하며 비밀번호를 확인해 볼 생각이다.

예를 들어 비밀번호 P가 123 이고 주어지는 번호 K가 100 일 때, 100부터 123까지 24번 확인하여 비밀번호를 맞출 수 있다.

P와 K가 주어지면 K부터 시작하여 몇 번 만에 P를 맞출 수 있는지 알아보자.


[입력]

입력으로 P와 K가 빈 칸을 사이로 주어진다.

[출력]

몇 번 만에 비밀번호를 맞출 수 있는지 출력한다.
