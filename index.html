풀이

BFS를 이용해서 풀었다.

  • 여기서 관건은 1의 위치를 찾아내서 BFS를 실행하는 것이다.
    • 1의 위치를 찾아내는 것 또한 BFS를 이용했고, find_one 함수 내에도 자체 visited 배열인 visited_one을 만들어야 무한 루프 도는 것을 막을 수 있다.

[프로그래머스] 여행경로 - DFS/BFS (파이썬)

2020-11-14
  • 문제 링크 : https://programmers.co.kr/learn/courses/30/parts/12421 > 여행경로

문제

주어진 항공권을 모두 이용하여 여행경로를 짜려고 합니다. 항상 ICN 공항에서 출발합니다.

항공권 정보가 담긴 2차원 배열 tickets가 매개변수로 주어질 때, 방문하는 공항 경로를 배열에 담아 return 하도록 solution 함수를 작성해주세요.

제한사항
  • 모든 공항은 알파벳 대문자 3글자로 이루어집니다.
  • 주어진 공항 수는 3개 이상 10,000개 이하입니다.
  • tickets의 각 행 [a, b]는 a 공항에서 b 공항으로 가는 항공권이 있다는 의미입니다.
  • 주어진 항공권은 모두 사용해야 합니다.
  • 만일 가능한 경로가 2개 이상일 경우 알파벳 순서가 앞서는 경로를 return 합니다.
  • 모든 도시를 방문할 수 없는 경우는 주어지지 않습 니다.
입출력 예
tickets return
[[ICN, JFK], [HND, IAD], [JFK, HND]] [ICN, JFK, HND, IAD]
[[ICN, SFO], [ICN, ATL], [SFO, ATL], [ATL, ICN], [ATL,SFO]] [ICN, ATL, ICN, SFO, ATL, SFO]
입출력 예 설명

예제 #1

[ICN, JFK, HND, IAD] 순으로 방문할 수 있습니다.

예제 #2

[ICN, SFO, ATL, ICN, ATL, SFO] 순으로 방문할 수도 있지만 [ICN, ATL, ICN, SFO, ATL, SFO] 가 알파벳 순으로 앞섭니다.


풀이

  • DFS에 사용되는 스택은 선입후출이기 때문에 조건 만일 가능한 경로가 2개 이상일 경우 알파벳 순서가 앞서는 경로를 return 합니다. 을 만족하기 위해서 역 알파벳 순으로 in 되어야 한다. = tickets.sort(reverse = True)
  • dict() 를 이용해 출발 공항 : 도착 공항1, 도착 공항2 와 같이 담아준다. (예제 2의 경우에는 'ICN' : 'SFO', 'ATL' 와 같이 담긴다.)
  • 스택(st)에 최초 출발 공항인ICN을 담아준다.
  • st[-1] : -1 은 배열의 마지막 요소를 의미 한다.
  • A 공항에서 출발하는 티켓이 없거나, A 공항에서 갈 수 있는 공항을 이미 간 경우 (routes[A]가 비어있는 경우) ans에 스택(st)에서 마지막으로 들린 공항부터 담는다.
    • A 공항에서 출발하는 티켓이 존재하거나, A 공항에서 갈 수 있는 공항이 있는 경우 (else), A 공항의 routes에 있는 도착 공항을 스택(st)에 담는다(그 공항에 도착했다는 의미).
  • 마지막으로 들린 공항부터 ans에 담았기 때문에 ans.reverse() 한 것이 답이 된다.

[프로그래머스] 단어 변환 - DFS/BFS (파이썬)

2020-11-11
  • 문제 링크 : https://programmers.co.kr/learn/courses/30/parts/12421 > 단어 변환

문제

문제 설명

두 개의 단어 begin, target과 단어의 집합 words가 있습니다. 아래와 같은 규칙을 이용하여 begin에서 target으로 변환하는 가장 짧은 변환 과정을 찾으려고 합니다.

예를 들어 begin이 hit, target가 cog, words가 [hot,dot,dog,lot,log,cog]라면 hit -> hot -> dot -> dog -> cog와 같이 4단계를 거쳐 변환할 수 있습니다.

두 개의 단어 begin, target과 단어의 집합 words가 매개변수로 주어질 때, 최소 몇 단계의 과정을 거쳐 begin을 target으로 변환할 수 있는지 return 하도록 solution 함수를 작성해주세요.

제한사항
  • 각 단어는 알파벳 소문자로만 이루어져 있습니다.
  • 각 단어의 길이는 3 이상 10 이하이며 모든 단어의 길이는 같습니다.
  • words에는 3개 이상 50개 이하의 단어가 있으며 중복되는 단어는 없습니다.
  • begin과 target은 같지 않습니다.
  • 변환할 수 없는 경우에는 0를 return 합니다.
입출력 예
begin target words return
hit cog [hot, dot, dog, lot, log, cog] 4
hit cog [hot, dot, dog, lot, log] 0
입출력 예 설명

예제 #1
문제에 나온 예와 같습니다.

예제 #2
target인 cog는 words 안에 없기 때문에 변환할 수 없습니다.


풀이

DFS를 이용하여 풀었다.

  • 맨 처음 target이 words 배열에 없는 경우를 체크하고 이런 경우 변환 될 수 없기 때문에 0을 반환 한다.

  • 앞에서 비교하면서 한글자만 다른 경우 answer += 1 을하고 stack에 해당 단어를 넣고 break 했다.

  • 앞에서부터 비교하기 때문에 문제의 예제 #1 같은 경우 dog-> log -> cog 로 변환될 경우가 있다. 이를 방지하기 위해 매번 target과도 한글자만 다른 경우가 있는지 검사해서 한글자만 다른경우 answer += 1 한 후 answer을 반환한다.

  • visited 배열을 이용해서 이미 변환에 사용된 단어가 또 비교되는 경우가 없도록 했다.

어려웠던 점

  1. 단어를 비교하는 방법 : 처음에 단어가 한 개의 알파벳만 다른 경우를 어떻게 체크하지 라고 생각했는데 단순히 앞에서부터 한알파벳씩 비교하면서 cnt를 세주면 됐다.. 생각보다 단순했다!
  2. ‘최소’의 경우를 구하는 법 : 앞에서부터 모든 경우를 살펴보지만 매번 target과 한글자만 차이나는지를 비교하기 때문에 최소 값을 구할 수 있었다.

[프로그래머스] 피보나치 수 (파이썬)

2020-10-27

문제링크 : https://programmers.co.kr/learn/challenges?tab=all_challenges > 피보나치 수

문제

피보나치 수는 F(0) = 0, F(1) = 1일 때, 1 이상의 n에 대하여 F(n) = F(n-1) + F(n-2) 가 적용되는 수 입니다.

예를들어

  • F(2) = F(0) + F(1) = 0 + 1 = 1
  • F(3) = F(1) + F(2) = 1 + 1 = 2
  • F(4) = F(2) + F(3) = 1 + 2 = 3
  • F(5) = F(3) + F(4) = 2 + 3 = 5

와 같이 이어집니다.

2 이상의 n이 입력되었을 때, n번째 피보나치 수를 1234567으로 나눈 나머지를 리턴하는 함수, solution을 완성해 주세요.

제한 사항

* n은 1이상, 100000이하인 자연수입니다.

입출력 예
n return
3 2
5 5
입출력 예 설명

피보나치수는 0번째부터 0, 1, 1, 2, 3, 5, … 와 같이 이어집니다.


풀이

먼저 재귀로 풀었다. 예전에도 피보나치 수의 경우 재귀로 풀면 재귀 안에 재귀가 두개 들어있어서 바이너리 트리와 같이 되고, 시간 복잡도가 커진다고 봤던 것 같다. 실제로도 테스트6까지만 성공하고 나머지는 런타임 에러와 시간초과가 났다.

찾아보니 재귀는 O(2^n)의 시간 복잡도를 갖고, 반복문을 이용할 때는 O(n)의 시간복잡도를 갖는다고 한다.

아래는 반복문을 이용해서 푼 풀이이다.

n이 5인 경우

a b num
0 1 0
1 2 1
2 3 2
3 5 3
5 8 4

이런식으로 while문이 진행된다. while문이 끝난 후 a 값을 1234567로 나눈 나머지 값이 답이 된다.

2020-10-25

문제

트럭 여러 대가 강을 가로지르는 일 차선 다리를 정해진 순으로 건너려 합니다. 모든 트럭이 다리를 건너려면 최소 몇 초가 걸리는지 알아내야 합니다. 트럭은 1초에 1만큼 움직이며, 다리 길이는 bridge_length이고 다리는 무게 weight까지 견딥니다.
※ 트럭이 다리에 완전히 오르지 않은 경우, 이 트럭의 무게는 고려하지 않습니다.

예를 들어, 길이가 2이고 10kg 무게를 견디는 다리가 있습니다. 무게가 [7, 4, 5, 6]kg인 트럭이 순서대로 최단 시간 안에 다리를 건너려면 다음과 같이 건너야 합니다.

경과 시간 다리를 지난 트럭 다리를 건너는 트럭 대기 트럭
0 [] [] [7,4,5,6]
1~2 [] [7] [4,5,6]
3 [7] [4] [5,6]
4 [7] [4,5] [6]

따라서, 모든 트럭이 다리를 지나려면 최소 8초가 걸립니다.

solution 함수의 매개변수로 다리 길이 bridge_length, 다리가 견딜 수 있는 무게 weight, 트럭별 무게 truck_weights가 주어집니다. 이때 모든 트럭이 다리를 건너려면 최소 몇 초가 걸리는지 return 하도록 solution 함수를 완성하세요.

제한 조건
  • bridge_length는 1 이상 10,000 이하입니다.
  • weight는 1 이상 10,000 이하입니다.
  • truck_weights의 길이는 1 이상 10,000 이하입니다.
  • 모든 트럭의 무게는 1 이상 weight 이하입니다.
입출력 예
bridge_length weight truck_weights return
2 10 [7,4,5,6] 8
100 100 [10] 101
100 100 [10,10,10,10,10,10,10,10,10,10] 110

풀이

풀이 과정

전체적인 풀이과정은 truck_weight에 있는 모든 트럭이 다리에 올라타면 첫번째 while문이 끝나고 , 트럭이 다리를 모두 건너면(다리 위 트럭 무게 총합이 0이면) 두번째 while문이 끝난다.

  • answer: 지나간 시간
  • bridge: 현재 다리 상태 (다리 길이만큼 인덱스를 갖는다.)
  • now_weight: 현재 다리에 올라가있는 트럭 무게들의 총합
  • truck : 트럭을 건너지 않은 트럭들 중 가장 앞에 있는 트럭
