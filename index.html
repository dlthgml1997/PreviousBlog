JAVA 언어 개요 - 특징

2021-01-01

자바 언어의 탄생

1991년

Green Project에서 가정용 전자기기에서 사용되는 작은 컴퓨터 언어를 디자인 했다.

이때,

  1. 작고 견고한 언어, 신뢰할 수 있는 코드
  2. 특정 디바이스에 종속되지 않아야 함

와 같은 특성을 고려해 Virtual Machine에서 동작하는 중간코드를 생성해 이식성이 높은 언어를 디자인 했다.

1992년

초기 자바 언어는 절차지향언어와 객체지향 혼용해 사용했던 언어인 C++ 를 기반으로 개발되었다. 그러다가 C++ 의 여러가지 한계에 부딪혀

완벽한 객체지향 언어인 Oak 가 개발되었고 제임스 고슬링에 의해 명명된 Oak는 Oak라는 프로그램이 존재했기 때문에 이후 JAVA로 변경 되었다.

1994년

World Wide Web의 발전과 Mosaic의 사용에 주목

자바를 기반으로 한 웹 브라우저 제작 (HotJava 브라우저로 발전함)

applet이라 불리는 자바의 기능을 보여주기 위해 브라우저가 중간코드(bytecode)를 해석할 수 있도록 만듦

1996년~ 현재

Netscape 2.0에서부터 자바가 지원됨

많은 업체들이 자바를 지원하게 됨

자바 버전 1.2가 출시되면사. 자바 2 플랫폼으로 명명됨.


자바 언어의 특징

  1. 단순

    C++의 전위처리, 포인터 사용, 다중 상속, 구조체, goto문 사용, 연산자 중복 기능 등을 제외

    가비지 컬렉터에 의한 자동 메모리 관리로 할당된 메모리 해제를 신경쓰지 않아도 된다.

  2. 객체지향(object-oriented)

    ​ 언어라는 객체를 사용해서 프로그램을 작성하도록 하는 언어

    -> 재사용성을 높일 수 있음.

    코드의 재사용을 넘어 객체의 재사용을 뜻하며, 프로그램 개발에 필요한 수많은 클래스 라이브러리들을 API로 제공함

  3. 분산 처리(Distributed)에 용이하다.

    분산 환경에서 TCP/IP 등의 프로토콜을 통해 효율적으로 실행할 수 있도록 설계된 언어이다.

    • TCP/IP 네트워크 기능 내장
    • HTTP/ FTP 등과 같은 프로토콜을 쉽게 사용할 수 있도록 라이브러리 제공
    • 서로 다른 컴퓨터 상의 객체들도 원격으로 호출하여 실행할 수 있는 원격 메서드 호출과 관련된 RMI(Remote Method Invocation)기능의 라이브러리 제공

  4. 인터프리터에 의해 실행됨

    .java 형태의 텍스트 파일 -> 컴파일러에 의해 자바가상머신에서 수행될 수 있는 중간코드로 컴파일된다. -> .class(클래스 파일 == 바이트 코드) -> 실제 실행을 위해 인터프리터에 의해 기계어로 변환 -> 컴퓨터에서 실행

    자바는 컴파일 언어임과 동시에 인터프리터 언어이다.

  5. 견고

    에러의 원인이 되는 요소들을 제거

    • 포인터를 사용하지 않음
    • 가비지 컬렉션
    • 데이터 타입 검사를 통해 에러 초기 검사
    • 실행시간에 발생하는 에러 초기

  6. 안전

    • 컴파일 시에 엄격하게 데이터 타입을 검사함으로써 프로그램 실행 시 발생할 수 있는 비정상적인 상황을 미리 감지

  7. 플랫폼 독립(Architecture Neutral)

    사용하는 운영체제나 CPU 등의 하드웨어 사양에 관계없이 실행될 수 있음.

    이식성이 높다.

  8. 높은 성능

    가비지 컬렉션 : 메모리가 할당된 객체의 상태를 추적 관리, 더이상 참조 되지 않으면 다종으로 할당된 메모리 해제 (불필요한 메모리 낭비 방지)

  9. 멀티스레드

    스레드 : 독자적으로 수행하는 하나의 작은 프로그램

    멀티스레드: 한번의 여러개의 스레드가 동시에 수행되는 과정 / 하나의 CPU가 여러 개의 프로그램을 동시에 수행하도록 함 (수행 속도를 빠르게 함)

  10. 동적

    변화하는 환경에 잘 적응되도록 설계됨

    기존의 프로그램에 영향을 주지 않고, 라이브러리에 새로운 메서드나 속성들을 추가할 수 있음

    프로그램과 프로그램에서 사용되는 라이브러리와의 연결을 실행시간에 수행함

    -> 라이브러리의 변화를 곧바로 적용할 수 있음

클라우드 컴퓨팅의 종류

어떤 자원을 제공하는지에 따라 구분한다.

1. SaaS (Software as a Service)

: 클라우드 서비스 중 가장 일반적인 것이다. 서비스 제공자가 모든 인프라와 소프트웨어 제품을 제공한다.

주요 특징

  1. 사용자는 웹 기반의 프론트엔드로 서비스와 상호작용한다.
  2. 필요할 때 비용만 내면 어디서든 곧바로 사용이 가능하다.
  3. 소프트웨어 설치를 위한 비용과 시간이 불필요하다.
  4. 중앙에서 해당 소프트웨어를 관리하기 때문에 사용자가 일일이 업그레이드나 패치 작업을 할 필요가 없다.

ex> 구글 앱스, 세일즈포스닷컴, MS 오피스365, 드롭박스와 같은 클라우드 스토리지 서비스 등


2. PaaS (Platform as a Service)

: 서비스 제공자가 자체 하드웨어 인프라에서 호스트하는 소프트웨어와 제품 개발 도구를 제공한다.

주요 특징

  1. 서비스 제공을 위해 개발 기술 모듈화, 표준화 능력, 기반에 최적화해 설치하는 기술력, 지원력 등의 기술요건이 필요하다.
  2. 모니터링이나 프로비저닝, 비용 처리방식(Billing)이 솔루션 개발사에 유리하게 구성되어 있다.

ex> Google App Engine, 마이크로소프트 Azure


3. IaaS (Infrastructure as a Service)

: 가상 서버라고도 불리며, 데이터 스토리지 및 데이터베이스와 같은 일련의 빌딩블록에 대한 엑세스를 제공한다.

주요 특징

  1. 서비스 제공을 위해 대규모 데이터 센터의 서버 구축 및 운용 능력, 자본력, 넓은 대지, 풍부한 전력 환경 등이 필요하다.

  2. 가상공간의 서버 자원을 제공하는 서비스로 서버의 확장이 가능하며 윈도우나 리눅스 모두 지원하므로 몇 분 안에 서버를 만들어 낼 수 있다.

  3. 사용자는 이러한 빌딩 블록을 결합하거나 계층 구조화하여 애플리케이션을 실행하는데 필요한 인프라를 구축할 수 있다.

ex> AWS(Amazon Web Services)와 Rackspace

[클라우드 컴퓨팅] 클라우드 컴퓨팅의 개념, 장/단점

2020-12-17

4차 산업혁명 시대가 도래하면서 수많은 컴퓨팅 자원과 데이터의 저장이 필요해졌고, 이로 인해 클라우드 컴퓨팅의 중요성이 점점 커지고 있으며 필수 불가결한 자원이 되고 있다.

클라우드 컴퓨팅이란?

저장소 공간 또는 CPU주기와 같은 리소스를 대여, 사용한 부분에 대해서만 비용 지불하는 것

이러한 서비스를 제공하는 회사를 “공용 클라우드 공급자”라고 한다. (ex> 마이크로소프트, 아마존, 구글)

클라우드 공급자는 작업을 실행하는데 필요한 실제 하드웨어에 대한 책임이 있으며, 이 하드웨어를 최신 상태로 유지해야 한다.

장점

  • 저비용, 고성능의 정보처리 기능을 활용할 수 있다.

  • 시스템 구축 및 개발 기간을 단축하고 필요한 경우 확장 가능한 효율성과 편의성도 제공한다.


클라우드 컴퓨팅 서비스 주요 기능

  1. 컴퓨팅 기능

    컴퓨팅 자원이라고도 하며, Windows 및 Linux 가상 머신을 의미

  2. 저장소 기능

    데이터나 개체의 저장소 기능

  3. 응용 프로그램 기능

    NoSQL 또는 SQL 데이터베이스 응용프로그램 등

  4. 네트워킹 기능

    가상 머신용 가상 네트워크 설정 등

  5. 분석 기능

    원격 분석 데이터 및 성능 데이터시각화 등


클라우드 배포 모델 종류

  1. 공용 클라우드 (Public Cloud)

    : 가장 일반적인 배포 모델, 모든 항목이 공급자의 하드웨어에서 실행된다.
    다른 클라우드 사용자와 컴퓨팅 리소스를 공유할 시 금액을 절감할 수 있다.
    ex> 아마존, 구글, 마이크로소프트

  2. 사설 클라우드 (Private Cloud)

    : 기업의 고유한 데이터 센터에 클라우드 환경 조성, 조직의 사용자에게 계산 리소스에 대한 셀프 서비스 액세스를 제공한다.
    이는 사용자에게 공용 클라우드의 시뮬레이션을 제공하지만, 대상 기업은 제공하는 하드웨어, 소프트웨어 서비스의 구매 및 유지 관리에 대한 전적인 책임을 진다.

  3. 하이브리드 클라우드

    : 공용 및 사설 클라우드 결합, 가장 적합한 위치에서 응용 프로그램 실행 가능
    ex> 공용 클라우드의 경우 웹사이트를 호스팅하고, 사설 클라우드의 경우 보안 수준이 높은 데이터베이스 연결에 사용한다. (의료 데이터와 같이 법적인 이유로 클라우드에 포함할 수 없는 몇가지 항목이 있는 경우에 유용)


클라우드 배포 모델에서 고객과의 상호작용 방법을 결정하려면?

  • 데이터 저장 위치, 응용프로그램 다운로드 방법 및 실행 위치 등에 따라서 고객과 상호 작용하는 방법을 정의
  • 자제 인프라를 관리하려는 의지 또는 관리해야 하는 필요성에 따라 달라진다.

리소스를 보호하는 책임

​ 사용자와 클라우드 공급자 간에 공유된다.

* 클라우드 공급자 : 도구를 제공하여 보안 위협 완화
* 클라우드 사용자 : 도구를 사용하여 리소스 보호

클라우드 컴퓨팅의 장점

  1. 종량제 가격 책정 모델을 제공한다.

    : 하드웨어를 임대하고 사용하는 리소스에 대한 요금만 지급한다.

  2. 요구 사항에 따라 수직적 및 수평적 크기 조정 모두 지원한다.

    • 수직적 크기 조정: 기존 서버의 능력을 늘리기 위해 리소스를 추가하는 프로세스
      ex> 더 많은 CPU 추가, 더 많은 메모리 추가

    • 수평적 크기 조정: 하나의 단위로 함께 작동하는 더 많은 서버를 추가하는 프로세스

      ex> 들어오는 요청을 처리하는 하나의 서버를 포함했다면, 추가하여 두 개를 포함

  3. 수요가 급증하는 등의 워크로드가 변경 되면 자동으로 리소스를 추가 또는 제거하여 보정이 가능하다.

    : 응용 프로그램을 빌드하고 배포하는 중요한 작업에 집중이 가능하다.

  4. 비즈니스를 운영하는 동안 데이터가 항상 준비되어 있음을 확신할 수 있다.

    : 클라우드 공급자는 데이터 백업, 재해/복구, 데이터 복제 등의 서비스를 제공해 데이터가 안전하게 유지될 수 있도록 한다.


클라우드 컴퓨팅 도입 시 고려사항 (단점)

  1. 기술적으로 완벽하지 않은 부분 존재

    ex> 고도의 신뢰성과 안정성이 요구되는 자율 주행 자동차의 경우, 짧은 네트워크 지연이나 작은 데이터 전송 오류라도 치명적인 사고로 이어질 수 있다. (이로 인해 데이터 발생 직전에 정보 처리 하는 엣지, 포그 컴퓨팅이 주목받고 있다.)

  2. 보안 위협 및 해킹에 대한 불안

  3. 비용 절감 측면에서 생각보다 적은 효과

[프로그래머스] 가장 큰 수 (파이썬)

2020-12-13
  • 문제링크 : https://programmers.co.kr/learn/courses/30/lessons/42746

풀이

이 문제에서 중요한 점은 [6,10,2]가 주어진 경우에 단순한 sort로는 1062가 되어 문제를 해결할 수 없다는 것이다.

프로그래머스 다른사람의 풀이에서 코드를 가져왔다.

  • sort 함수를 통해 기존 숫자들의 길이* 3 한 수를 key로 해서 내림차순 정렬을 한다. (1000이하의 수이기 때문에 *3을 해준다.)
  • [0,0,0]의 경우 정수는 000이 아닌 0이기 때문에 이를 위해 int로 변환했다가 다시 str로 변환한다.

새로운 정렬 방법을 알 수 있는 문제였다.


참고할만 한 좋은 풀이

함수를 이용해서 정렬을 하고자할 때는 아래 방법을 사용하면 된다.

1을 반환하면 a가 앞으로 -1을 반환하면 a가 b뒤로 정렬된다
