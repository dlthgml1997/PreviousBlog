[boj] 2564:경비원 - 구현 (java)

2021-02-08
  • 문제 링크: https://www.acmicpc.net/problem/2564

풀이

  • 스토어와 동근이의 위치는 테두리에만 존재하기 때문에 1,2,3,4 (동서남북) 여부와 왼쪽, 위쪽으로부터의 위치가 주어지면 실제 좌표 위치를 알 수 있다.

  1. 동근이와 경비해야할 store가 서, 동에 위치하는 경우

    • Math.abs(dongPos.y - store.y) 가 M이 라는 것은 서로 동, 서쪽으로 평행하다는 것을 의미한다.

    • 이때 거리는 위로 가는 경우(회색), 아래로 가는 경우(빨간색) 중 거리가 더 가까운 값에 M 을 더한 값이다.

  2. 동근이와 경비해야할 store가 북, 남에 위치하는 경우

    • Math.abs(dongPos.x - store.x) 가 N이 라는 것은 서로 북, 남쪽으로 평행하다는 것을 의미한다.

    • 이때 거리는 동근이가 오른쪽로 가는 경우(회색), 왼쪽으로 가는 경우(빨간색) 중 거리가 더 가까운 값에 M 을 더한 값이다.

  3. 동근이와 경비해야할 store가 평행하지 않는경우

    • 좌표끼리 뺀값의 절대값을 더한 값이 거리가 된다.

[안드로이드] Corutine 간단 정리(미완성)

2021-02-05

코루틴이 뭘까 ? (Corutine)

코틀린 뿐만 아니라 파이썬, Javascript 등 여러언어에서 지원하고 있는 개념 !

새로운 개념이 아닌 초창기부터 존재하던 개념이다.

근데 왜 지금 공부하는 걸까?

코루틴을 사용하면 비동기 처리가 너무나도 쉽게 이루어질 수 있다.


코루틴 키워드 3가지

  1. 협력형 멀티 태스킹
  2. 동시성 프로그래밍 지원
  3. 비동기 처리를 쉽게 도와줌

1. 협력형 멀티 태스킹 (Co + rutine)

Co 는 협력, 함께 라는 의미를 지니고 있다.

rutine은 하나의 태스크, 함수

즉, 코루틴은 협력하는 함수이다.

여기서 rutine은 main/ sub로 나누어지는데 흔하게 볼 수 있는 main 함수와 sub 함수 같은 개념으로 이해하면 편하다.

일반 함수는 루틴에 진입점(함수 호출)과 탈출점(return이나 닫는 괄호’}’)이 명확하다. 코루틴은 진입점과 탈출점이 여러개이다.

suspend 로 선언된 함수를 만나면 코루틴 밖으로 잠시 나갈 수 있다.

[boj] 2493: 탑 (java) / 스택

2021-02-04
  • 문제 링크: https://www.acmicpc.net/problem/2493

풀이

  1. 오른쪽 탑에서부터 왼쪽으로 레이저 신호를 보내기 때문에 마지막 인덱스(오른쪽)부터 살펴본다(신호를 보낸다). stack에는 레이저 신호를 보냈지만 신호를 받은 탑이 등장하지 않은 탑들이 담긴다.

    이때, 주의할 점은 stack에는 탑의 번호(i)를 집어넣었다. answer 배열의 인덱스가 될 것이기 때문이다.

    answer 배열은 인덱스에 해당하는 탑의 레이저 수신호를 받은 탑의 번호가 담겨져야한다.

  2. i번째 탑이 스택 값들보다 높이가 높거나 같으면 스택 값들을 빼고, 낮으면 그냥 add 한다.

  • 스택 = 신호를 보낸 탑이 들어감 (아직 신호를 받은 탑은 없는 상태)

  • i번째 탑 = 스택 안에 있는 탑보다 높이가 높거나 같으면 이들의 레이저 신호를 받은것을 의미한다.

    • i번째 탑의 높이보다 작거나 같은 스택값은 모조리 pop(신호를 받은 탑이 생겼다!!!! 하면서 나온다, 선입후출)한다. 신호를 보낸 탑의 번호인 stack.pop()은 answer의 인덱스가 되고, i+1을 answer에 넣으면 된다. (i+1인 이유는 탑의 번호가 1부터 시작하기 때문이다 )
    • i번째 탑의 인덱스는 스택에 무조건 add (i번째 탑의 신호를 받게될 탑은 다음 for문 이후로 나올것이기 때문 : remind! 신호는 오른쪽에서 왼쪽으로 보낸다)

  • 스택에 값이 남았다는 것은 이들의 신호를 아무 탑도 받지 못했다는 것을 의미한다. (이유: 가장 왼쪽 탑이거나, 왼쪽 탑들보다 높이가 높은 경우)

    • 이땐 answer[인덱스]에 0을 할당해야하는데 answer 배열은 기본값이 0으로 초기화되어있기 때문에 그냥 끝낸다.
