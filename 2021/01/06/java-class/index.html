1) 클래스의 구조

  • 클래스 선언부

    • 접근 제한자(Access Modifier): 접근 제한 시 사용되는 키워드로 생략이 가능하다.
    • 클래스(Class): class라는 예약어로 선언하며 클래스명은 식별자 이름 규칙에 따라 명시한다.

  • 클래스 몸체에는 클래스의 멤버가 선언된다.

    • 생성자(Constructor): 객체의 초기화를 담당한다.
    • 멤버 변수(Variable): 클래스가 가지는 속성을 정의한다.
    • 메서드(Method): 클래스가 가지는 데이터를 조작하고 변환한다.

  • 클래스 구조 예시


접근 제한자 (Access Modifier)

접근 관련 modifier를 통해 객체지향 언어의 중요 특성 중 하나인 정보 은닉(Information Hiding)을 구현할 수 있다.

  • 접근 권한 예약어에는 public, protected, private, (default) 가 있다.

    • public : 모든 클래스에서 접근이 가능한 클래스임을 의미한다.
    • (default) : 접근 권한 예약어를 지정하지 않으면, 같은 패키지 내의 클래스에서만 접근이 가능하다.

  • 활용 방법 예약어에는 final,abstract, static, transient, volatile, syncronized, native 가 있다.

    • final : 자식 클래스를 가질 수 없는 클래스임을 의미한다.
    • abstract : 객체 생성이 불가능한 추상클래스를 의미한다.

소프트웨어 개념을 다이어그램으로 그리기 위해 사용하는 시각적 표기법

  • 멤버 변수의 선언

    변수의 종류

    • 전역(Global) 변수 : 클래스 선언부 밑에 선언된 변수로 멤버 변수라고 칭한다. 여러 메서드에서 공통으로 사용할 수 있다.
    • 지역(Local) 변수 : 메서드 선언부 밑에 선언된 변수로 해당 변수가 선언된 메서드 내에서만 사용할 수 있다.

    여기서 modifier를 통해 변수에 대한 접근 권한이나 활용 방법을 제어할 수 있다. 미지정시, 패키지 내의 클래스에만 접근이 가능한 객체 변수라는 의미를 갖게 된다.

    멤버 변수에 사용되는 Modifier - 접근 권한

    • public : 모든 클래스에서 접근이 가능
    • protected: 동일 패키지에 속하는 클래스와 하위 클래스 관계의 클래스에 의해 접근이 가능
    • private: 변수가 선언된 클래스 내에서만 접근이 가능

    멤버 변수에 사용되는 Modifier - 활용 방법

    • final: 변수를 상수로 이용하는 경우 사용한다.
    • static: 클래스에 소속된 클래스 변수를 의미하며, 일반적으로 클래스 변수라고 한다.

3) 메서드 (Method)

메서드를 정의함으로써 코드가 간결해지고 재사용성이 높아진다.

3-1) 메서드 Modifier

메서드 Modifier를 이용해 적절하게 정보 숨김을 구현할 수 있다.

접근 권한을 나타내는 Modifier의 경우 그 쓰임이 멤버 변수와 동일하다.

  • 메서드 Modifier - 접근 권한
    • public : 모든 클래스에서 접근이 가능하다.
    • protected : 동일 패키지에 속하는 클래스와 하위 클래스 관계의 클래스에 의해 접근이 가능하다.
    • private : 클래스 내에서만 접근이 가능하다.
  • 메서드 Modifier - 활용 방법
    • final : Overriding 이 불가능한 메서드를 정의할 때 이용한다.
    • static : 클래스에 소속된 클래스 메서드를 의미하며, 클래스 생성 시 만들어진다.
    • abstract : 추상 메서드를 의미하며, 하위 클래스에 의해 구현된다.
    • syncronyzed: Thread의 동기화를 위한 메서드이다.

3-2) 메서드의 자동 구현

  • 정보 은닉의 기본

    • 멤버 변수들은 대부분의 경우 private 로 선언한다.
    • public 으로 지정한 메서드를 통해 멤버 변수에 접근하도록 클래스를 구현한다.

  • 이클립스에서는 private 변수에 대한 public Getter, Setter 메서드들을 자동으로 생성할 수 있는 기능을 제공한다.

    • Getter 메서드

      private 변수에 저장된 값을 리턴하는 getXXX() 메서드를 의미한다.

    • Setter 메서드

      private 변수에 값을 저장하는 getXXX() 메서드를 의미한다.

  • 이클립스에서 Getter / Setter 메서드 자동 구현

    1. private으로 접근 제한된 변수들을 가진 클래스 정의

    2. 작성된 소스에서 <Alt> + <Shift> + <S> 키를 누른 후, 셋톱 박스에서 Generate Getters and Setters를 누른다.

      위와 같은 메서드가 클래스 안에 생성된다.

  • toString() 메서드 자동 구현

    toString() 메서드는 생성된 객체의 상태를 쉽게 확인 가능하게 한다.
